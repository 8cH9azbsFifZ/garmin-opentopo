Een korte toelichting op het gebruik:

Stel je hebt binnen de huidige dir een directory 'nltiles' met de Nederlandse OSM
tiles en een directory 'cycletiles' met de Nederlandse Cycle map tiles, dan kan de
conversie zo plaats vinden:

gmapi-builder.py -t nltiles/11000000.tdb -b nltiles/11000000.img nltiles/*img

Het maakt dan een nieuwe directory (onder de huidige dir) genaaamd 'OSM NL Normal.gmapi'
met daarbinnen de gehele structuur zoals de Mac-applicaties die wensen. Als een
Mac gebruiker deze dir kopieert kan hij in Finder dubbelklikken om het onder Roadtrip
te installeren.

Voor de cyclemap gaat het analoog:

gmapi-builder.py -t cycletiles/12000000.tdb -b cycletiles/12000000.img cycletiles/*img

Je moet de baseimage dus in feite twee keer meegeven als argument: een keer om
aan te geven dat het de baseimage is, en nog een keer in de lijst van
img-bestanden die uitgepakt moeten worden.

Met -o kun je eventueel een andere basedirectory als uitvoer geven (dan wordt
onder die directory 'OSM NL Normal.gmapi' gemaakt); maar die directory moet wel
bestaan. Als er al een 'OSM NL Normal.gmapi' bestaat onder de uitvoer directory
wordt die eerst verwijderd.

Met -v of -vv kun je eventueel nog meer uitvoer op je scherm krijgen; en een
style file (.typ) kan met -s opgenomen worden. De onderdelen van routable maps
worden ook ondersteund volgens berichten van gebruikers.

En als laatste: met -d (dryrun) wordt er niets daadwerkelijk weggeschreven, en worden
alleen alle bestanden geparsed.

Meer informatie and informatie over de grafische versie kan gevonden worden op
de OSM-wiki:
http://wiki.openstreetmap.org/wiki/Gmapibuilder
